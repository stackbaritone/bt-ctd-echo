Aide — Gérer les modèles (mode simple)
Info
Vos modifications sont sauvegardées en brouillon local (dans votre navigateur). Elles apparaissent automatiquement dans l'app principale en quelques secondes. Cliquez « 🚀 Publier » pour les partager avec tous les utilisateurs.

1. Modifier un modèle existant

  1. Sélection — Dans la colonne de gauche, cliquez sur la tuile du modèle.
  2. Identifiant — Le champ ID identifie le modèle de façon unique (lettres, chiffres, underscore).
  3. Catégories — Choisissez une catégorie existante dans les menus déroulants ou tapez une nouvelle dans les champs texte. Les catégories apparaissent dans l'app principale pour filtrer les modèles.
  4. Titre et Description — Complétez en français et en anglais. Le titre apparaît dans la liste des modèles, la description donne le contexte d'utilisation.
  5. Objet et Corps — Rédigez l'objet de l'email et son corps dans les deux langues.
  6. Variables (placeholders) — Insérez des variables avec <<NomVariable>> dans l'Objet/Corps. Exemples: <<ClientName>>, <<ProjectNumber>>, <<NewDeadline>>. N'ajoutez PAS de suffixe _FR ou _EN dans le modèle (le système gère ça automatiquement).
  7. Sync variables — Après avoir modifié le texte, cliquez « Sync variables » pour mettre à jour la liste détectée.
  8. Descriptions des variables — En bas de l'éditeur, complétez les descriptions FR/EN et valeurs par défaut pour chaque variable détectée. Cela aide les utilisateurs à comprendre quoi remplir.
  9. Sauvegarde — Les changements sont enregistrés automatiquement en brouillon local. Pour publier définitivement, cliquez « 🚀 Publier » ou « Télécharger JSON » pour sauvegarder localement.

2. Supprimer un modèle

  1. Sélection — Cliquez sur le modèle dans la liste de gauche.
  2. Action — Cliquez le bouton « Supprimer » en haut à droite de l'éditeur.
  3. Une confirmation apparaît - validez pour supprimer définitivement du brouillon.
  4. Le modèle disparaît immédiatement de la liste.
  5. Important: La suppression ne touche que votre brouillon local. Pour que le modèle disparaisse aussi de l'app principale, vous devez cliquer « Publier sur GitHub ».
  6. Si vous supprimez par erreur, cliquez « Réinitialiser » pour restaurer la version officielle (avant vos modifications).

Astuce: Si vous supprimez le dernier modèle d'une catégorie, pensez à cliquer « Nettoyer catégories » pour retirer la catégorie vide des menus.

3. Ajouter un nouveau modèle

  1. Création — Dans la colonne de gauche (« Modèles »), cliquez « Ajouter un modèle ».
  2. ID — Saisissez un identifiant simple (lettres, chiffres, underscore). Exemple: welcome_client.
  3. Complétez les champs FR et EN (Titre, Description, Objet, Corps).
  4. Placeholders — Insérez les variables nécessaires: <<ClientName>>, <<ProjectNumber>>, etc. Elles sont détectées automatiquement.
  5. Vérifiez l’orthographe, accents, cohérence FR/EN.
  6. Publiez via « Publier » → Télécharger JSON pour transmettre le nouveau modèle, ou utilisez « Exporter Excel » si vous préparez plusieurs ajouts.

Besoin d’aide pour décrire les variables ? Ajoutez leur nom puis revenez plus tard compléter les descriptions.

4. Travailler avec Excel (import/export)

📤 Exporter vers Excel

  1. Cliquez « Exporter Excel » pour télécharger tous vos modèles actuels.
  2. Le fichier .xlsx contient un seul onglet « Templates » avec toutes les colonnes nécessaires.
  3. Format des colonnes: ID, CATEGORY_EN, CATEGORY_FR, DESCRIPTION_EN, DESCRIPTION_FR, TITLE_EN, TITLE_FR, TEMPLATE_EN, TEMPLATE_FR, VARIABLES_DESCRIPTION_EN, VARIABLES_DESCRIPTION_FR
  4. Important: Dans l'Excel exporté, les variables ont des suffixes _EN et _FR (ex: <<ClientName_EN>>). C'est normal pour le format d'échange Excel.

📥 Importer depuis Excel

  1. Cliquez « Importer » et sélectionnez votre fichier .xlsx.
  2. Le système détecte automatiquement s'il s'agit d'un fichier Excel ou JSON.
  3. Une boîte de dialogue apparaît:
    • ✓ OUI (Fusionner) — Ajoute les nouveaux modèles sans toucher aux existants (recommandé)
    • ✗ NON (Remplacer) — Supprime TOUT et remplace par le contenu du fichier (attention!)
  4. Après l'import, vérifiez les modèles dans la liste de gauche.
  5. Les variables sont automatiquement extraites et ajoutées à la bibliothèque.
  6. N'oubliez pas de publier après vérification!

📋 Télécharger un modèle Excel vierge

Cliquez « Modèle XLSX » pour obtenir un fichier exemple avec les bonnes colonnes et un modèle de démonstration.

Astuce Excel: Vous pouvez modifier plusieurs modèles à la fois dans Excel, puis les réimporter en mode Fusionner. C'est pratique pour des corrections en lot ou l'ajout massif de nouveaux modèles.

5. Publication et sauvegarde

💾 Sauvegarde automatique (brouillon local)

Tous vos changements sont automatiquement enregistrés dans le localStorage de votre navigateur. C'est un brouillon qui persiste entre les sessions, mais:

  • ⚠️ Il n'est PAS partagé avec d'autres utilisateurs ou appareils
  • ⚠️ Il peut être perdu si vous videz le cache du navigateur
  • ✅ Il vous permet de travailler progressivement sans perdre vos modifications

📤 Télécharger JSON

Cliquez « Télécharger JSON » pour sauvegarder votre brouillon complet sur votre ordinateur.

  • Génère un fichier complete_email_templates.json
  • Contient tous les modèles, variables, catégories et métadonnées
  • Peut être réimporté plus tard avec « Importer »
  • Utile pour faire une copie de backup

🚀 Publier sur GitHub

Cliquez « 🚀 Publier » pour envoyer vos changements directement dans le dépôt GitHub et mettre à jour l'application principale.

Ce qui se passe lors de la publication :

  • Le fichier complete_email_templates.json est mis à jour sur GitHub
  • L'application principale se synchronise automatiquement avec la nouvelle version
  • Tous les utilisateurs voient les changements immédiatement après rechargement
  • L'indicateur de statut passe à « À jour »

Bonne pratique : Vos modifications apparaissent dans l'app principale en quelques secondes (synchronisation locale). Publiez sur GitHub quand vous êtes satisfait pour les partager avec tous les utilisateurs.

6. Gérer les catégories

🎨 Couleurs des catégories

Cliquez « Catégories » pour ouvrir le gestionnaire de couleurs.

  • Chaque catégorie a une couleur par défaut qui apparaît dans l'app principale
  • Vous pouvez personnaliser les couleurs pour mieux les différencier
  • Un aperçu montre le rendu dans l'application
  • Cliquez « Enregistrer » pour appliquer les changements

🧹 Nettoyer les catégories inutilisées

Cliquez « Nettoyer catégories » pour supprimer les catégories qui n'ont plus aucun modèle associé.

  • Le système trouve automatiquement les catégories orphelines
  • Une confirmation liste les catégories qui seront supprimées
  • Les catégories disparaissent des menus déroulants et de l'app principale
  • N'oubliez pas de publier après le nettoyage!

7. Types et modes des modèles

📧 🤖 📝 Type de modèle

Le menu déroulant Type de modèle définit la nature du contenu.

  • 📧 Courriel (valeur par défaut) — Modèle de courriel complet avec un objet et un corps. Apparaît dans l'app principale avec le champ Objet actif.
  • 📝 Texte court (blurb) — Fragment de texte réutilisable (signature, clause, explication courte). L'objet n'est pas affiché dans l'app principale.
  • 🤖 Prompt — Instruction pour un assistant IA (Copilot, ChatGPT, etc.). L'objet n'est pas utilisé ; le corps contient le prompt.

Le type est visible dans l'app principale par un badge coloré sur la tuile du modèle. Seuls les types blurb et prompt affichent ce badge (les courriels n'ont pas de badge de type).

👥 Modes (utilisateurs)

Les cases à cocher Modes définissent quels groupes d'utilisateurs voient ce modèle dans l'app principale.

  • 👥 Conseillers (teal) — Rôle par défaut. Visible par tous les conseillers de traduction.
  • 🔐 Gestion (ambre) — Réservé aux gestionnaires. Accessible via le code gestion dans l'app principale.
  • 👔 Équipe Admin (bleu) — Pour l'équipe administrative. Accessible via le code equipe_admin.
  • 🤝 Fournisseurs (violet) — Pour les relations avec les fournisseurs. Accessible via le code relations_fournisseurs.

Si aucune case n'est cochée, le modèle est automatiquement attribué au mode conseillers. Vous pouvez cocher plusieurs modes simultanément pour rendre un modèle visible à plusieurs groupes.

8. Outils de rédaction

🏗️ Bouton Squelette

Cliquez 🏗️ Squelette pour préremplir automatiquement les corps FR et EN selon la catégorie du modèle.

  • Le squelette est un brouillon bilingue avec des <<variables>> déjà intégrés — il suffit de remplacer les crochets [ ] par votre contenu réel.
  • Si le corps est déjà rempli, une confirmation apparaît avant d'écraser.
  • Squelettes disponibles pour 8 catégories : devis et approbations, suivis et annulations, documents et mise en forme, délais et livraison, clarifications et instructions, sécurité et droits d'auteur, insatisfaisant, fin d'année financière.
  • Si aucun squelette n'est défini pour la catégorie sélectionnée, un message vous en informe.

📋 Bouton Dupliquer

Cliquez 📋 Dupliquer pour créer une copie exacte du modèle sélectionné.

  • Un nouvel ID est généré automatiquement (id_copie, id_copie_1, etc.).
  • Les titres FR/EN reçoivent le suffixe « (copie) » / « (copy) ».
  • Vous êtes automatiquement redirigé vers le nouveau modèle pour l'éditer.
  • Pratique pour créer une variante d'un modèle existant sans recommencer à zéro.

➕ Bouton Ajouter manquantes

Cliquez ➕ Ajouter manquantes pour enregistrer automatiquement dans la bibliothèque toutes les variables utilisées dans ce modèle qui n'y sont pas encore.

  • Le système détecte tous les <<placeholders>> dans l'Objet et le Corps (FR et EN).
  • Les variables déjà présentes dans la bibliothèque globale ne sont pas modifiées.
  • Les nouvelles variables sont ajoutées avec des descriptions vides — complétez-les ensuite dans la section Variables en bas de l'éditeur.
  • Un message indique combien de variables ont été ajoutées (ou « Aucune variable à ajouter » si tout est déjà enregistré).

9. Exporter en Word

Cliquez 📄 Exporter Word dans le menu Import / Export pour générer un document Word professionnel contenant tous vos gabarits.

Option 1 — Langue

  • 🇫🇷 + 🇬🇧 Bilingue (par défaut) — Français et anglais côte à côte dans chaque section.
  • 🇫🇷 Français — Corps et objet en français seulement.
  • 🇬🇧 English — Body and subject in English only.

Option 2 — Public cible

  • 📋 Tous les modèles (par défaut) — Toutes catégories et tous modes confondus.
  • 👥 Conseillers — Modèles attribués au mode Conseillers.
  • 🔐 Gestion — Modèles réservés aux gestionnaires.
  • 👔 Équipe Admin — Modèles pour l'équipe administrative.
  • 🤝 Relations fournisseurs — Modèles pour les fournisseurs externes.

Contenu du document généré

  • Page de garde avec le nombre de modèles, de catégories et la date de génération.
  • Table des matières avec liens cliquables vers chaque modèle.
  • Modèles organisés par catégorie (un par page) avec l'objet et le corps prêts à copier-coller.
  • Les variables <<placeholders>> sont surlignées en jaune dans le document.
  • Index alphabétique des mots-clés en annexe.

Le fichier généré s'appelle CTD_Catalogue_Modeles_[langue]_[date].docx. Il est téléchargé directement dans votre dossier Téléchargements.

❓ Questions fréquentes (FAQ)

Q: J'ai ajouté un modèle hier mais il n'apparaît pas aujourd'hui. Pourquoi?

R: Vous avez probablement oublié de cliquer « Publier sur GitHub » ou « Télécharger JSON ». Les changements restent en brouillon local (localStorage) jusqu'à publication. Si vous avez vidé le cache du navigateur ou changé d'appareil, le brouillon est perdu.

Q: Mes variables n'apparaissent pas dans la liste en bas.

R: Cliquez le bouton « Sync variables » après avoir modifié le texte. Le système détecte automatiquement tous les <<placeholders>> dans l'Objet et le Corps.

Q: Dois-je mettre _FR et _EN dans mes variables?

R: NON! Dans la console admin, utilisez seulement <<ClientName>>, <<ProjectNumber>>, etc. Les suffixes _FR/_EN sont ajoutés automatiquement lors de l'export Excel. Ne les ajoutez jamais manuellement dans vos modèles!

Q: J'ai un ID en doublon lors de l'import Excel.

R: Le système ajoute automatiquement un suffixe numérique (ex: welcome_email_1) pour éviter les conflits. Renommez l'ID après import si nécessaire.

Q: Comment supprimer une catégorie qui apparaît encore dans le menu?

R: Supprimez d'abord tous les modèles de cette catégorie, puis cliquez « Nettoyer catégories ». Ensuite, publiez sur GitHub pour que le changement soit visible dans l'app principale.

Q: Que fait le bouton "Réinitialiser"?

R: Il efface votre brouillon local (localStorage) et recharge la version officielle depuis le fichier JSON. Attention: Tous vos changements non publiés seront perdus! Utilisez-le seulement si vous voulez recommencer à zéro.

Q: Puis-je travailler à plusieurs sur les modèles?

R: Oui, mais attention aux conflits! Chaque personne travaille sur son propre brouillon local. Lorsque quelqu'un publie sur GitHub, les autres doivent cliquer « ⟳ Recharger » pour récupérer la dernière version. Coordonnez-vous pour éviter d'écraser les changements des autres!

Q: Comment savoir si mes changements sont publiés?

R: L'indicateur de statut en haut de la page affiche « À jour » (vert) si votre brouillon correspond à la version publiée, ou « Non publié » (orange) si vous avez des modifications locales non publiées.

Q: Combien de temps sont conservés les changements en brouillon local?

R: Indéfiniment, tant que vous ne videz pas le cache du navigateur. Mais pour la sécurité, exportez régulièrement un JSON de backup ou publiez sur GitHub.

Q: Quelle est la différence entre les types Courriel, Blurb et Prompt?

R: Courriel est le type standard avec un objet et un corps. Blurb est un texte court réutilisable (signature, clause, note) qui n'a pas d'objet. Prompt est une instruction pour un assistant IA ; seul le corps est utilisé. Dans l'app principale, les blurbs et prompts affichent un badge coloré sur leur tuile pour les distinguer des courriels.

Q: Comment fonctionne le contrôle des modes (conseillers, gestion, etc.)?

R: Par défaut, un modèle est visible par tous les Conseillers. En cochant d'autres modes, vous le rendez aussi visible aux groupes correspondants. Les utilisateurs de l'app principale doivent entrer le code du mode (ex: « gestion ») dans le champ prévu pour voir les modèles restreints. Un modèle peut appartenir à plusieurs modes simultanément.

Q: Le bouton Squelette ne fait rien. Pourquoi?

R: Le bouton Squelette n'est actif que si la catégorie du modèle correspond à l'une des 8 catégories prédéfinies (devis, suivis, documents, délais, clarifications, sécurité, insatisfaisant, fin d'année financière). Pour les autres catégories, aucun squelette n'est disponible et un message vous en informe. Assurez-vous d'avoir sélectionné un modèle avec une catégorie reconnue.

Astuce : vous pouvez garder cette fenêtre ouverte pendant que vous travaillez.
