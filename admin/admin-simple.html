📚 Guide d'utilisation – Console d'administration ECHO

🚀 Prise en main rapide

  1. Sélectionner un modèle dans la colonne de gauche ou créer un nouveau avec le bouton "Ajouter un modèle"
  2. Modifier les champs : ID, catégories (FR/EN), titres, descriptions, objets et corps
  3. Utiliser le formatage riche : sélectionner le texte dans les champs Corps pour activer la barre d'outils (gras, italique, couleurs, etc.)
  4. Ajouter des variables : taper <<nom_variable>> directement dans le texte – elles seront détectées automatiquement
  5. Enregistrer localement : les modifications sont sauvegardées automatiquement comme brouillon dans votre navigateur
  6. Publier sur GitHub : cliquer "Publier sur GitHub" pour déployer les changements en production

✨ Formatage riche du texte

Les champs Corps (FR) et Corps (EN) supportent le formatage HTML complet.

  • Sélectionner du texte pour afficher la barre d'outils flottante
  • Formatage de base : gras, italique, souligné, barré
  • Titres : H1, H2, H3 pour structurer le contenu
  • Listes : listes à puces ou numérotées
  • Couleurs :
    • Cliquer sur le swatch de couleur pour ouvrir le menu déroulant
    • Choisir parmi 8 couleurs prédéfinies ou utiliser l'input personnalisé
    • Applicable pour la couleur du texte et le surlignage
  • Variables préservées : les <<variables>> restent fonctionnelles même avec formatage

Important : Le formatage est stocké en HTML dans le JSON et s'affiche automatiquement dans l'application web principale.

📝 Variables dynamiques

  • Syntaxe : <<nom_variable>> (ex: <<nom_client>>, <<numero_dossier>>)
  • Détection automatique : les variables sont extraites des champs Objet et Corps
  • Noms conseillés : lettres minuscules, chiffres et underscores uniquement (ex: date_echeance)
  • Section Variables :
    • Affiche toutes les variables détectées
    • Permet d'ajouter des valeurs d'exemple (FR et EN)
    • Bouton "Sync variables" pour forcer la mise à jour de la liste
  • Bilingue : chaque variable peut avoir des exemples différents en français et en anglais

💾 Gestion des données

Importer

• Formats acceptés : JSON ou Excel (.xlsx)
• Télécharger le modèle XLSX vide pour voir la structure attendue
• L'import remplace le brouillon actuel (attention aux modifications non publiées)

Exporter Excel

• Génère un fichier .xlsx contenant tous les modèles
• Utile pour backup, révision hors-ligne ou collaboration
• Peut être réimporté après modifications

Télécharger JSON

• Télécharge le fichier JSON complet avec tous les modèles
• Format utilisé par l'application web et GitHub
• Inclut le timestamp de dernière mise à jour

⟳ Recharger

• Récupère la version publiée sur GitHub
• Écrase le brouillon local – toutes modifications non publiées seront perdues

🚀 Publier sur GitHub

• Déploie les modifications sur GitHub et met à jour l'app web
• Met à jour automatiquement le timestamp de publication
• Les utilisateurs de l'app web voient les changements immédiatement après rechargement

🎨 Catégories et organisation

  • Catégories bilingues : chaque modèle doit avoir une catégorie FR et EN (peuvent être identiques ou traduites)
  • Sélection rapide : les dropdowns affichent les catégories existantes pour cohérence
  • Nouvelles catégories : laisser le dropdown sur "-- Nouvelle catégorie --" et taper dans le champ texte
  • Couleurs des catégories :
    • Bouton "Catégories" pour définir une couleur par catégorie
    • Affichage visuel dans la liste de modèles pour identification rapide
  • Nettoyer catégories : supprime les catégories non utilisées de la liste globale

🔐 Sécurité et accès

  • Authentification requise : mot de passe demandé à l'ouverture de la console
  • Session persistante : restez connecté tant que vous ne vous déconnectez pas manuellement
  • Accès depuis l'app principale : bouton Admin discret en bas à gauche (visible pour tous, protégé par mot de passe)
  • Brouillon local : toutes les modifications sont sauvegardées dans le navigateur (localStorage)
  • Synchronisation locale : vos changements apparaissent dans l'app principale en quelques secondes (sans publier)
  • Publication GitHub : pour partager vos modifications avec tous les utilisateurs
  • Indicateur de statut : badge "À jour" ou "Non publié" indique si vos changements sont publiés

⚠️ Bonnes pratiques

  • Publier régulièrement pour éviter de perdre du travail en cas de fermeture accidentelle
  • Tester dans l'app web après publication pour vérifier le rendu des formatages
  • Utiliser des ID uniques : lettres, chiffres et underscores, pas d'espaces ni caractères spéciaux
  • Nommer clairement les variables pour que les utilisateurs comprennent leur fonction
  • Exporter en Excel avant modifications majeures comme backup de sécurité
  • Éviter de recharger depuis GitHub si vous avez des modifications non publiées
  • Confirmer avant suppression : un modal demande confirmation avec le nom du modèle – action irréversible

🐛 Dépannage

Les variables ne s'affichent pas correctement

• Cliquer sur "Sync variables" pour forcer la détection
• Vérifier la syntaxe : <<nom_variable>> avec deux chevrons de chaque côté
• Pas d'espaces à l'intérieur des chevrons

Le formatage ne s'affiche pas dans l'app web

• S'assurer d'avoir publié sur GitHub (les brouillons locaux ne sont pas visibles publiquement)
• Les utilisateurs doivent rafraîchir la page pour voir les nouveaux changements
• Le cache du navigateur est automatiquement ignoré lors du chargement des templates

Erreur lors de la publication GitHub

• Vérifier votre connexion internet
• Réessayer après quelques instants
• En cas d'échec répété, utiliser "Télécharger JSON" pour sauvegarder vos modifications localement

Mes modifications ont disparu

• Vérifier si quelqu'un a publié sur GitHub ou cliqué sur "⟳ Recharger" (écrase le brouillon local)
• Les brouillons sont liés au navigateur – changer d'ordinateur ou de navigateur recharge depuis GitHub
• Toujours publier sur GitHub pour sauvegarder définitivement
